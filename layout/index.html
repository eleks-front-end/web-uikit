Grid system

Web UI Kit provides a powerful mobile-first grid system for building layouts of all shapes and sizes based on flexboxes. It’s based on a 12-column layout and has multiple tiers, one for each media query range. You can use it with Sass mixins or our predefined classes.

How it works

At a high level, here’s how the grid system works:

  • There are three major components—containers, rows, and columns.
  • Containers—.e-container for fixed width or .e-container-fluid for full width—center your site’s contents and help align your grid content.
  • Rows are horizontal groups of columns that ensure your columns are lined up properly.
  • Content should be placed within columns, and only columns may be immediate children of rows.
  • Column classes indicate the number of columns you’d like to use out of the possible 12 per row. So if you need three equal-width columns, use .e-col-sm-4.
  • Column widths are set in percentage, so they’re always fluid and sized correspondingly to their parent element.
  • Columns have horizontal padding to create the gutters between individual columns.
  • There are five grid tiers, one for each responsive breakpoint: extra small, small, medium, large, and extra large.
  • Grid tiers are based on minimum widths, meaning they apply to that one tier and all those above it (e.g., .e-col-sm-4 applies to small, medium, large, and extra large devices).
  • You can use predefined grid classes or Sass mixins for more semantic markup.

Variables

By using variables, you can determine the number of columns, the gutter width, and the media query point at which floating columns begin. We use these to generate the predefined grid classes documented above, as well as for the custom mixins listed below.

Predefined classes

In addition to our semantic mixins, Web UIKit includes an extensive set of prebuilt classes for quickly creating grid columns. It includes options for device-based column sizing, reordering columns, and more.

Example: Column wrapping

If more than 12 columns are placed within a single row, each group of extra columns will, as one unit, wrap onto a new line.

.e-col-xs-9
.e-col-xs-4
Since 9 + 4 = 13 > 12, this 4-column-wide div gets wrapped onto a new line as one contiguous unit.
.e-col-xs-6
Subsequent columns continue along the new line.

Example: Offsetting columns

Move columns to the right using .e-col-md-offset-* classes. These classes increase the left margin of a column by * columns. For example, .e-col-md-offset-4 moves .e-col-md-4 over four columns.

.e-col-md-4
.e-col-md-4 .e-col-md-offset-4
.e-col-md-3 .e-col-md-offset-3
.e-col-md-3 .e-col-md-offset-3
.e-col-md-6 .e-col-md-offset-3

Example: Nesting columns

To nest your content with the default grid, add a new .e-row and set of .e-col-sm-* columns within an existing .e-col-sm-* column. Nested rows should include a set of columns that add up to 12 or fewer (it is not required that you use all 12 available columns).

Level 1: .e-col-sm-9
Level 2: .e-col-xs-8 .e-col-sm-6
Level 2: .e-col-xs-4 .e-col-sm-6

Example: Column ordering

Easily change the order of our built-in grid columns with .e-col-md-push-* and .e-col-md-pull-* modifier classes. Also you can use classes .e-col-md-first or .e-col-md-last.

.e-col-md-9 .e-col-md-push-3
.e-col-md-3 .e-col-md-pull-9
.e-col-md-9
.e-col-md-3 .e-col-md-first
