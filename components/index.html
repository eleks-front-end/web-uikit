Validation

Web UIKit includes validation styles for danger, warning, and success states on form controls. To use, add .e-has-warning, .e-has-danger, or .e-has-success to the parent element. Any .e-input-label, .e-input, and .e-text-muted within that element will receive the validation styles.

Buttons

Use custom button styles for actions in forms, dialogs, and so on. Our framework supports contextual variations, sizes, states, and more.

Example

ELEKS Web UI Kit contains six predefined button styles, and each of them serves its own semantic purpose.

Inverse buttons

If you need a button without any coloring, replace the default modifier classes with the .e-btn-*-outline ones to remove all background images and colors on any button.

Disabled buttons

Make buttons look inactive by adding the disabled boolean attribute to any <button> element.

Buttons sizing

If you need larger or smaller buttons, add .e-btn-lg or .e-btn-sm for additional sizes.
